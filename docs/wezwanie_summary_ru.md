# Что уже реализовано по Wezwanie

1. **Автоматическое чтение Wezwanie при загрузке**
   - Если сотрудник загружает документ типа "Wezwanie", файл разбирается на лету.
   - Из текста извлекаются номер дела и дата сдачи отпечатков, чтобы не вносить их вручную.

2. **Заполнение данных клиента**
   - Найденный номер дела сохраняется в карточке клиента, если он отличается от текущего.
   - Найденная дата отпечатков записывается в поле `fingerprints_date` и отображается в карточке.

3. **Письмо клиенту о недостающих документах**
   - После загрузки Wezwanie формируется список документов, которых нет в чеклисте клиента.
   - Клиенту автоматически уходит письмо с этим списком и просьбой донести отсутствующие пункты.

4. **Шаблоны и логи**
   - Текст письма хранится в шаблоне `clients/email/missing_documents.txt`, чтобы его было легко редактировать.
   - Отправка писем ведётся через стандартный механизм `send_mail`; ошибки логируются для диагностики.

5. **Покрытие тестами**
   - Есть unit-тесты на разбор Wezwanie, обновление данных клиента и отправку письма о недостающих документах.

Эти шаги автоматизируют заполнение ключевых полей после получения Wezwanie и сокращают время на напоминания клиентам о документах.
