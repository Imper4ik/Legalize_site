# clients

## Назначение
`clients` — ключевое бизнес-приложение проекта для ведения клиентов, документов, платежей, напоминаний и рабочих коммуникаций.

## Функциональные блоки
- Клиенты: карточка, CRUD, заметки, статусы, печатные формы.
- Документы: загрузка, верификация, чеклисты требований, парсинг Wezwanie.
- Платежи: учёт оплат, суммы, статусы, сроки.
- Напоминания: документные и платежные дедлайны.
- Почта: массовые и индивидуальные отправки, предпросмотр, логирование.
- Метрики и калькулятор финансовых требований.

## Структура
- `models/` — доменные модели (`Client`, `Document`, `Payment`, `Reminder`, `EmailLog`, `ServicePrice`, `Company`).
- `views/` — HTTP-слой (разбит по доменам: clients/documents/payments/reminders/emails/metrics/admin).
- `services/` — бизнес-логика без привязки к UI.
- `templatetags/` — кастомные фильтры/теги.
- `management/commands/` — фоновые и административные команды.
- `templates/clients/` — страницы интерфейса.

## Точка входа
Маршруты приложения подключены под префиксом `/staff/` (через `clients.urls`).
