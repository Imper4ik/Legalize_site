Translation Studio — инструкция (обновлённая)

Коротко
--------
Translation Studio — инструмент для удобного редактирования переводов прямо на сайте и в отдельном дашборде. Поддерживает сразу три языка: RU, EN, PL. Изменения сохраняются в PO-файлы и автоматически компилируются в MO.

Как открыть
-----------
- В админ-панели (в выпадающем меню по вашему имени) нажмите `Translation Studio`.
- Прямой адрес: `/studio/dashboard/`.
- Быстрая однократная активация: добавьте `?studio` в адресную строку (например, `/clients/123/?studio`) — это включит ин‑context режим только для текущего запроса.

Два режима работы
-----------------
1) Дашборд (3-панельный вид)
- Видно msgid и все три языка в строке (RU/EN/PL) — удобно сравнивать и массово править.
- Поиск по ключу (msgid) или по тексту — строка поиска сверху.
- "Save Row" — сохранить одну строку.
- "Save All Changes" — сохранить все помеченные строки сразу.
- При сохранении PO-файлы обновляются, а система вызывает `compilemessages`, поэтому изменения становятся видны почти мгновенно.

2) Studio Mode — редактирование на сайте (in-context)
- Включите: в дашборде нажмите кнопку `Studio Mode: OFF` → она переключится в `Studio Mode: ON` (зелёный).
- Либо используйте `?studio` в URL, если хотите временно включить режим для одного запроса.
- Перейдите на любую страницу сайта (например, профиль клиента).
- Наводите курсор на текст: строки, которые можно редактировать, подсвечиваются (синяя рамка).
- Горячая клавиша: Ctrl + Shift + Клик на подсвеченном тексте — откроется окно редактирования со всеми тремя языками.
- Нажмите "Save & Compile" — изменения сохраняются и ребилдятся автоматически.

Важные детали и исправления
---------------------------
Я внёс(ла) несколько улучшений и исправлений, чтобы решить проблему, которую вы описали (строки отображались как `[[i18n:...]...]` и не открывались для редактирования):

- Горячая клавиша изменена на `Ctrl + Shift + Click` (Alt+Click конфликтует в некоторых браузерах).
- Серверная "обёртка" (wrapping) перевода теперь выполняется глубже (в `translations.apps.TranslationsConfig.ready`) и дополнена — теперь покрываются также формы множественного числа (`ngettext`) — это помогает захватить больше случаев, где перевод мог пропасть.
- Чтобы поймать тексты, которые всё ещё не были обернуты на сервере (частичные шаблоны, динамически подгружаемые блоки), добавлен клиентский «Deep Fix»:
  - Overlay JS теперь подгружает специальный API `/studio/scan-api/`, который отдаёт соответствия "переведённый текст → msgid" для текущего языка.
  - JS сканирует DOM и автоматически заменяет вхождения перевода на редактируемые `<span class="studio-editable" data-msgid="...">...</span>`.
- Обновлён middleware/template-injection: теперь скрипт overlay подключается, если включён флаг в сессии `studio_mode` ИЛИ если в URL есть `?studio` — это покрывает сценарий быстрого однократного включения.
- При сохранении вручную — флаг `fuzzy` автоматически убирается (как вы просили).

Краткий список изменений в коде
------------------------------
- `translations/apps.py` — расширена глубинная обёртка gettext, добавлена поддержка `ngettext` (plural).
- `translations/middleware.py` — оставлена основная замена маркеров на `<span>`, но теперь клиент может поймать пропущенные случаи.
- `translations/views.py` — добавлен `scan_translations_api` (путь `/studio/scan-api/`).
- `translations/static/translations/js/translation_overlay.js` — загружает mapping и агрессивно сканирует DOM, создавая `.studio-editable` спаны для найденных переводов.
- `templates/base.html` — скрипт подключается, если включён `studio_mode` или если в URL есть `?studio`.

Что делать, если что-то всё ещё не работает
------------------------------------------
1. Проверьте, что вы вошли как суперпользователь.
2. Откройте `/studio/dashboard/` и нажмите кнопку переключателя — убедитесь, что она стала зелёной `Studio Mode: ON`.
   - Если вы хотите временно включить режим для одной страницы, добавьте `?studio` в URL.
3. Перейдите на нужную страницу и нажмите `Ctrl + Shift + Click` на подсвеченном тексте.
4. Если подсветки нет, откройте консоль браузера (DevTools) и проверьте ошибки и логи:
   - Ищите сообщения `Studio scan map load failed` или `Studio scanAndWrap error` — это поможет понять, подгрузился ли `scan-api` и не упала ли обработка.
   - Проверьте, что в HTML есть `<script src="/static/translations/js/translation_overlay.js">` (для суперпользователя и при `studio_mode`).
5. Если видите на странице `[[i18n:...]]...[[/i18n]]` и при этом нет подсветки — попробуйте перезагрузить страницу с `?studio` в адресной строке (это обеспечит загрузку клиентского скрипта для текущего запроса).

Если нужно, я могу:
